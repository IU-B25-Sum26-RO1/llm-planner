# Голосовое управление роботом с LLM-планировщиком

Проект реализует систему голосового и текстового управления роботом, в которой большая языковая модель (LLM) разбивает высокоуровневую команду оператора на последовательность элементарных примитивов (предобученных навыков). Каждый примитив вызывает соответствующий предобученный блок управления: RL/IL-политику для захвата объектов в интерактивном режиме, специализированную политику для точных манипуляций (вставка, сборка) или классический контроллер для перемещений. Например, команда «возьми красный кубик и положи в коробку» автоматически разворачивается в последовательность: detect → move_to → grasp → move_to → place. Цель проекта: дать оператору естественный высокоуровневый способ управления роботом, опираясь при этом на библиотеку надёжных предобученных низкоуровневых навыков для непосредственного исполнения.

## Виртуальный стенд (UR10e Gazebo Simulation)

Базовая сцена для проекта Embodied AI. Включает в себя манипулятор UR10e, зафиксированный на столе, подвесную камеру Intel RealSense D435 (Eye-to-Hand), двухпалый захват, тестовые объекты (лоток и кубик) и настроенные контроллеры (`ros2_control`) для удержания позы и перемещения. 

Проект полностью контейнеризирован и оптимизирован для работы в **WSL2** с аппаратным ускорением графики.

### Быстрый запуск (WSL2 / Windows)

**1. Сборка Docker-образа**
Откройте терминал в корневой папке проекта и соберите образ (в него уже зашиты все необходимые зависимости ROS 2, плагины контроллеров и аудио-библиотеки):
```bash
docker build -t ur10e-gazebo-img .
```

**2. Запуск контейнера (с пробросом GPU)**
Для плавной работы в симуляции в 60 FPS на WSL 2 используется проброс DirectX графики.
```bash
docker run -it --rm \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  -v /usr/lib/wsl:/usr/lib/wsl \
  --device /dev/dxg \
  -e LD_LIBRARY_PATH=/usr/lib/wsl/lib \
  -v $(pwd):/workspace \
  ur10e-gazebo-img
```

**3. Запуск симуляции**
Оказавшись внутри контейнера (root@...), запустите launch-файл сцены. Благодаря автоматической сборке в Dockerfile, достаточно ввести короткую команду:
```bash
ros2 launch ur10e_scene start_scene.launch.py
```
