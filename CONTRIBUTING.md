# Работа с файлами

## Установка локальной копии

Для начала работы с файлами необходимо пройти первых пару шагов в [Гайде по установке и запуску](docs/deployment.md)

---

## Обновление локальной копии

Перед началом работы:

```bash
git switch main # переключаемся на main-ветку (локальную)
git pull origin main # подтягиваем изменения из origin (Github) в main (локально)
uv sync # синхронизируемся по pyproject.toml и uv.lock
```

---

## Работа с ветками

Создать новую ветку:

Вообще любые изменения проводить в новой ветке и пушить в неё же. Потом, после того, как вы закончили работу, запросить Pull Request.
**Название веток должно соответствовать названию задачи (issue) которое она закрывает в таком формате:**

```
<номер-задачи>-<сокращённое-название-задачи>
```
(сокращённое название задачи сокращает не сами слова, а количество слов)

Пример:

```bash
git switch -c 3-реформат-документации
```

После завершения работы:

```bash
git status # Какие файлы претерпели изменения
```

```bash
git add <путь_к_файлу> # добавление файлов по одному
# git add .  - Все измененные файлы
git commit -m "Описание изменений" 
# после добавления всех изменений, отправить на gh в указанную ветку
git push -u origin <номер-задачи>-<сокращённое-название-задачи>
```
**(Советуется делать коммиты по каждому файлу отдельно для удобного описания изменений каждого файла)**

Затем создать Pull Request в GitHub.

---

## Добавление новой зависимости

```bash
uv add <package>
```

Пример:

```bash
uv add fastapi
```

После добавления зависимости необходимо коммитить:

* pyproject.toml
* uv.lock


# Задачи (issue) и пулл реквесты

## Назначение задач

В большинстве случаев задачи назначаются участниками команды на самих себя, но в отдельных случаях возможно назначение другим участником.
При создании задачи необходимо **указать её тип**:
- Баг (красный)
- Фича (синий)
- Задание (жёлтый)
Когда задача закрывается пулл реквестом и тот был одобрен, необходимо закрыть и саму задачу (пометить как выполненной). В отдельных случаях задачи не требуют пулл реквест, поэтому они закрываются без него при выполнении.

Всё это назначается в гитхабе проекта в разделе *Issues*.

## Пулл реквесты

Наименование пулл реквестов такое же, как и в ветке гита, которая идёт на этот запрос.
В описании указать, немного обобщив если много текста, изменения и добавления функционала или документации.
Так же необходимо привязать задачу которая решается этим пулл реквестом и запросить ревью у одного из участников команды.

Всё это происходит так же в гитхабе в разделе *Pull Requests*.
